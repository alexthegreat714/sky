# Sky Logs Directory

This directory contains all log files generated by Sky agent.

## Log Files

### `sky.log`
Main application log with startup, errors, warnings, and general operations.

**Format:** Standard Python logging format with timestamp, level, and message.

**Example:**
```
2025-11-08 12:34:56,789 - Sky - INFO - Sky Agent initializing...
2025-11-08 12:34:56,790 - Sky - INFO - Loading configuration from config/sky.yaml
2025-11-08 12:34:56,791 - Sky - INFO - Identity loaded successfully
```

### `sky/actions.jsonl` (when created)
Audit trail of all actions taken by Sky in JSONL format.

**Example entry:**
```json
{
  "timestamp": "2025-11-08T12:34:56.789Z",
  "action": "tool_invocation",
  "tool": "morning_reporter",
  "status": "success",
  "duration_ms": 1234,
  "metadata": {
    "user": "alex",
    "reason": "scheduled_morning_briefing"
  }
}
```

## Log Rotation

Logs are rotated when they reach 10MB in size, keeping the 5 most recent files:
- `sky.log`
- `sky.log.1`
- `sky.log.2`
- `sky.log.3`
- `sky.log.4`

## Log Levels

- **DEBUG**: Detailed diagnostic information
- **INFO**: General informational messages
- **WARNING**: Warning messages for potentially problematic situations
- **ERROR**: Error messages for failures that don't stop the agent
- **CRITICAL**: Critical errors that may cause shutdown

## Viewing Logs

### Real-time monitoring
```bash
tail -f logs/sky.log
```

### Search for errors
```bash
grep ERROR logs/sky.log
```

### View action audit trail
```bash
cat logs/sky/actions.jsonl | jq
```

## Notes

- Log files are not committed to git (see `.gitignore`)
- Logs are essential for debugging and audit compliance
- Action logs may be used for constitutional accountability in future phases
